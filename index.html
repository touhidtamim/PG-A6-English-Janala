Frequently Asked Questions

What is the difference between var, let, and const?
  • var: Function-scoped or globally scoped. Re-declare othoba update kora jabe.
  • let: Block-scoped. Update kora jabe, kintu same scope re-declare kora jabe na.
  • const: Block-scoped. Update ba re-declare kora jabe na, kintu value object hole er vitorer properties change kora jay.
What is the difference between map(), forEach(), and filter()?
  • map(): Prottek element ke modify kore notun array return kore.
  • forEach(): Prottek element-e provided function ekbar kore run kore, kintu kono return dey na.
  • filter(): Provided function-e pass howa test-er upor base kore notun array banay, jekhane shudu test pass kora elements gulo thake.
What are arrow functions and how are they different from regular functions?
  • Arrow Functions: Choto & short syntax. Nijer "this" context nai, parent scope theke ney. Single-line function hole implicit return use kora jay
  • Regular Functions:Traditional function syntax. Nijer "this" context thake, ei "this" tar caller object ke refer kore. Function declaration/expressions dara lekha hoy.

How do closures work in JavaScript?

A closure is a function that remembers variables from its outer function, even after the outer function has finished. It keeps access to the outer function's scope.

How closures work:

  • Lexical Scope: A function remembers where it was created.
  • Outer Variable Access: Inner functions can access variables from the outer function even after it finishes.
  • Use Cases: Closures are used for data privacy, maintaining state, and creating private variables.
